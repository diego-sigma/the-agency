## [Unreleased]

### Added
- **`/earl-learn [repo]`** — new skill that teaches Earl from review comments on the user's own past PRs. Mines merged PRs authored by the current user in the named repo (defaults: every repo with an existing lessons file) from 2026-01-01 onward, identifies "accepted" review threads (resolved + no pushback reply), clusters them into themes, and writes the result to `<vault>/earl-lessons/<repo>.md`. Bootstrap on first run; incremental refresh thereafter via a sibling `<repo>.analyzed.json` manifest. Lessons are **repo-scoped, not project-scoped** — they live at the vault root (not inside any agency project) so they follow Earl across every project that touches the same repo. `CLAUDE.md` "How to use personas" updated to include the lessons file in every Earl spawn whose task touches a known repo with a lessons file.
- **Parker** — new PR-specialist persona (`templates/personas/parker.md`) and matching `/parker` skill. Drafts pull request descriptions matching the target repo's house style and pre-flights the diff for missed best practices. Backed by per-repo profiles in `templates/pr-author/<repo>.md` derived from analyzing the 500 most recently merged PRs (descriptions) + top 100 by review activity (best-practice critiques) per repo. Initial profiles ship for `sigmacomputing/slate` and `sigmacomputing/mono-node`, each with a sibling `<repo>.analyzed.json` manifest recording exactly which PR numbers were sampled so refreshes can be incremental. New routing-table row in `CLAUDE.md` directs "drafting a PR / pre-flighting a diff" tasks to Parker.
- `/pulse [N]` — globally-available tech-news triage skill. Searches Hacker News, Anthropic news, the `modelcontextprotocol` GitHub org, ArXiv, GitHub trending, Reddit, TLDR, and (optionally) X via a curated accounts list. Cross-references stories across sources, prioritizes by upvotes/likes/stars/comments, writes each as a paragraph summary into `<vault>/pulse/inbox/`, and presents them with a 0–3 rating prompt. User verdicts adjust per-topic and per-source weights (with 10 %/week decay) so future runs surface what matters. Auto-grows `<vault>/pulse/x-accounts.txt` when handles are mentioned across multiple articles. Excludes any project whose directory name starts with `oncall` when building topic seeds.
- `/pause [name]` — cancel the scheduled auto-gather cron job for a project (uses `CronDelete`). Manual `/gather-context` runs a catch-up gather AND recreates the cron at the end, so it doubles as resume.
- Background `/gather-context` skips when the user has been idle for over an hour, gated by a `~/.claude/.last-user-activity` marker file refreshed by a `UserPromptSubmit` hook (manual invocation always proceeds)
- Background `/gather-context` honors **quiet hours** (8 pm – 6 am local time) and **weekends** (Sat/Sun) — the cron is scheduled with `13 6-19 * * 1-5` so it never fires outside the active window, with a runtime hour/dow backstop in the skill. Manual `/gather-context` always proceeds regardless of time/day. `/link-project` now uses `CronCreate` directly instead of `/loop` to install the schedule.
- **Scheduler is project-scoped, not session-scoped.** A new lease file at `~/.claude/the-agency-sessions/<project>.scheduler` records which session owns the auto-gather for a project and a `last_heartbeat`. New `/link-project` invocations check the lease before scheduling; if another session already owns it and the heartbeat is fresh (≤24 h), the new session shares the data instead of adding a duplicate cron. Background gathers refresh the heartbeat; `/pause` and `/unlink-project` release the lease if owned. Stale leases (heartbeat older than 24 h) can be claimed by any session.
- `/link-project` installs the `UserPromptSubmit` hook in `~/.claude/settings.json` if missing
- Hero image at the top of `README.md` (`assets/social-preview-1280.jpg`, 1280×640); same file intended for the GitHub repo's social preview
- `/gather-context` resource refresh skips Google Drive body fetches when the doc's `modifiedTime` is unchanged — uses `get_file_metadata` as a cheap pre-check, requires `drive_file_id` and `drive_modified_time` in resource frontmatter
- `/link-project` surfaces a per-project MCP checklist (Slack / Atlassian / Google Drive / `gh`) computed from the project's `config.md` + saved resources, so users know which connectors to verify with `/mcp`
- README "Prerequisites" lists the Google Drive MCP and explains the per-project checklist

### Changed
- Parker always appends `_Generated by Claude Code using [The Agency](https://github.com/diego-sigma/the-agency)._` as the final line of every PR description he drafts. The footer is documented in both `templates/personas/parker.md` (step 6 of "How you draft a PR description") and `skills/parker.md` (step 4) so future personas/skills that touch PR drafting can match the convention.
- Auto-gather runs **once per weekday morning** instead of hourly. The cron is now `13 7 * * 1-5` (7:13 am local Mon–Fri) — a single daily gather rather than the previous hourly polling between 6 am and 7 pm. The runtime hour/weekend backstop in `/gather-context` is unchanged but is now largely redundant (kept for sessions still running the older hourly cron). The idle-skip threshold is bumped from **1 hour → 24 hours** so a user who steps away briefly before 7:13 am doesn't lose the day's gather; only multi-day absences (vacation, weekend trips) cause a skip. Manual `/gather-context` always proceeds and re-arms the cron, unchanged.


- Framework is now cwd-independent. Runtime state moved out of the repo:
  - `.the-agency-config` → `~/.claude/the-agency-config` (key=value: `vault=`, `repo=`)
  - `.the-agency-sessions/` → `~/.claude/the-agency-sessions/`
  - `scripts/init.sh` injects a managed block into `~/.claude/CLAUDE.md` that points at the framework repo, so agent routing and personas apply from any cwd
- All skills + framework `CLAUDE.md` updated to reference the new global paths
- Re-running `./scripts/init.sh` migrates legacy in-repo files automatically (idempotent)
- "When making changes" maintenance checklist in `CONTRIBUTING.md` (skill changes, persona changes, vault structure changes, agent context changes)
- "Framework maintenance rules" section in `CLAUDE.md` reminding Claude to keep README, CHANGELOG, and CLAUDE.md in sync with code changes

## [0.1.0] - 2026-04-24

### Added
- Initial framework release
- Four agent personas: Steve (Staff Engineer), Earl (Senior Engineer), Debug Bot 500 (Code Reviewer), Pat (Project Manager)
- Per-project Obsidian vault with tiered knowledge storage (live → daily → weekly)
- Wiki-based agent context with `wiki/activity.md` as the volatile primary reference
- Slash commands: `/new-project`, `/link-project`, `/unlink-project`, `/update-project`, `/gather-context`, `/status`, `/daily-digest`, `/explain`, `/team-review`, `/todo`
- Incremental context gathering via `last_gathered` timestamp
- Background auto-refresh loop (silent, hourly)
- Resource auto-fetch from shared URLs with source-of-truth updates
- Session notes, plans, and per-project preferences
- Agent routing by task type (code changes → Earl, reviews → Steve + Debug Bot 500, plans → Earl + Steve, project questions → Pat)
